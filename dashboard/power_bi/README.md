# Power BI Build Specification

Power BI is the primary BI dashboard deliverable for SocialLens BI. Build the report from the final remote PostgreSQL warehouse whenever possible.

## Data Source

Preferred source:

- PostgreSQL database from `.env`
- Current remote workflow database: `nexabi`
- Schema: `social_dw`
- Connector: Power BI PostgreSQL database connector

Views:

- `social_dw.vw_executive_overview`
- `social_dw.vw_daily_engagement`
- `social_dw.vw_sentiment_trend`
- `social_dw.vw_content_performance`
- `social_dw.vw_competitor_benchmark`
- `social_dw.vw_posting_time_heatmap`
- `social_dw.vw_viral_posts`

Backup source:

- CSV exports in `dashboard/exports/`
- These are generated by:

```powershell
python -m etl.cli export --database-url $env:DATABASE_URL
```

Do not build the submitted dashboard from sample-only development data.

## Refresh Contract

Before opening Power BI, run the final ETL sequence:

```powershell
python -m etl.cli run --sources youtube --no-sample-fallback --database-url $env:DATABASE_URL
python -m etl.cli quality --database-url $env:DATABASE_URL
python -m etl.cli export --database-url $env:DATABASE_URL
```

The ETL must use a real `YOUTUBE_API_KEY`. The final run should fail instead of silently switching to sample data.

## Pages

1. Executive Overview
   - KPI cards: total posts, total reach, total engagement, average engagement rate, sentiment ratio.
   - Line chart: daily reach and engagement from `vw_daily_engagement`.
   - Slicers: date, platform, brand.

2. Content Performance
   - Bar chart: content type performance from `vw_content_performance`.
   - Table: top posts from `vw_viral_posts`.
   - Scatter: reach vs engagement when using detailed post exports or SQL validation queries.

3. Sentiment Analysis
   - Trend chart: positive, neutral, and negative counts from `vw_sentiment_trend`.
   - Brand/platform slicers.
   - KPI: sentiment ratio from `vw_executive_overview`.

4. Competitor Benchmarking
   - Compare Highlands Coffee, Phuc Long, and The Coffee House where data is available.
   - Visuals: share of voice, engagement, average engagement rate, and sentiment.
   - Source: `vw_competitor_benchmark`.

5. Posting Time Heatmap
   - Matrix or heatmap by day of week and hour.
   - Source: `vw_posting_time_heatmap`.

## Model Guidance

- Import each view as a separate table unless DirectQuery is required for the demo.
- Keep the view names visible in the data model for traceability.
- Use date fields from the views for page-level slicers.
- Use brand/page and platform fields consistently across pages.
- Prefer simple measures in Power BI because the warehouse views already contain BI-ready aggregates.
